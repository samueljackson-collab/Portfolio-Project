## üìã PR Resolution Strategy

### Triage Your PRs Using This Framework:

| Category | Action | When to Use |
|----------|--------|-------------|
| **MERGE** | Merge immediately | Tests pass, no conflicts |
| **UPDATE** | Fix then merge | Minor issues to resolve |
| **SPLIT** | Break into smaller PRs | Too large/multiple concerns |
| **CLOSE** | Close without merge | Obsolete/superseded |
| **REBASE** | Rebase on main | Conflicts need resolution |
| **DEFER** | Postpone | Valid but not priority |

### Recommended PR Workflow:
1. List all open PRs across repos
2. Categorize each using framework above
3. Process MERGE category first (quick wins)
4. Batch UPDATE items together
5. Schedule time for SPLIT/REBASE work
6. Close obsolete PRs with explanation
